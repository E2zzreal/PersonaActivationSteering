# 2026-04-20 三阶段管道实验与 SCL 修复留档

## 实验概述

在 661 条 Claude SFT 数据上完成了 Stage1→Stage2→Stage3 完整管道训练和评估。
同时修复了 SCL（对比损失）的三层隐藏 bug 并验证了修复效果。

## 三阶段评估结果

| 模型 | 均分 | 分布 |
|------|---:|------|
| Stage2 (SFT + gate) | **3.033** | 1×2, 27×3, 2×4 |
| Stage1 (SFT only) | 3.000 | 1×2, 28×3, 1×4 |
| Stage3 (scl=0.1) | 3.000 | 30×3 |
| Stage3 (scl=1.0) | 2.967 | 1×2, 29×3 |

**结论**：Stage2 是最佳配置。Stage3 的 SCL 无论权重高低都无法提升分数。

## SCL 三层 bug 修复

1. **batch_size=1**：SCL 在 losses.py:96 直接返回 0 → 改为 batch_size=4
2. **user_ids 优先**：trainer 传 user_ids（全局唯一）给 SCL → 改为用 personality 字符串
3. **无负例**：grouped sampler 整 batch 同 personality → 改为混合 batch（2 personality × 2 samples）

修复后 SCL 产生非零值（~1.09），但 6 个 epoch 仅变化 0.003，梯度信号极弱。

## 根因分析

v_t 向量不分化：冻结 encoder 对不同 personality embedding 差异太小 + gate 12% 注入强度 + projection MLP 放大能力不足 → 所有 personality 的 v_t 几乎相同。

## DPO 实验（同日）

DPO 在 PersonaSteer 上结构性失效：80% 步骤 margin=0（模型和参考模型 logprobs 相同），因 12% 注入强度不足以产生 token 级差异。
