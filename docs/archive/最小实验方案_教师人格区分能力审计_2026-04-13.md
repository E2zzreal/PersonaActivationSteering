# 最小实验方案：教师人格区分能力审计（2026-04-13）

日期：2026-04-13
最后验证日期：2026-04-13
状态：待实施

## 1. 实验目的

确认候选教师模型是否适合用于后续 persona 数据合成。

判断标准不是“写得好不好”，而是：
- 是否真的能在不同 persona 之间拉开稳定差异；
- 是否能在自然、不泄露的前提下保留人格特征。

## 2. 最小实验设置

### 2.1 样本数
- user_input：`20`
- persona：`3`
- 每 persona 候选数：`2`

总候选数：`120`

### 2.2 persona 组
建议先用三组差异大的 persona：
- P1：外向、热情、社交导向
- P2：克制、细致、安静内省
- P3：理性、分析型、目标导向

### 2.3 输入来源
优先从现有验证集抽取：
- `data/split/val.jsonl`

这样可以和现有项目任务分布保持一致。

## 3. 生成阶段

每个 `user_input × persona` 组合：
- 让教师模型生成 `2` 个候选
- 候选都必须满足：
  - 面向用户的完成态回答
  - 禁止输出 thinking / planning
  - 不要自我分析草稿

## 4. 评审阶段

评审模型对候选做三类判断：
1. 哪个更符合目标 persona；
2. 两个候选差异是否足够明显；
3. 是否存在平均化 / thinking 泄露 / persona 偏移。

## 5. 输出产物

建议输出为一个 JSON 文件，字段包括：
- `sample_id`
- `user_input`
- `persona_id`
- `persona_text`
- `candidate_a`
- `candidate_b`
- `preferred`
- `difference_strength`
- `avg_collapse_flag`
- `leak_flag`
- `failure_tags`
- `judge_note`

## 6. 成功标准

若满足以下条件，则说明教师模型可进入下一步造数：
- thinking 泄露率 `< 5%`
- 可辨人格率 `> 60%`
- 差异明显率 `> 50%`
- 平均化率 `< 30%`

## 7. 后续分支

### 若通过
- 进入“生成正反例偏好数据”阶段
- 扩展到 `200~500` 对 preferred / rejected

### 若不通过
- 优先迭代教师 prompt / 评审 rubric
- 不建议直接大规模蒸馏

## 8. 一句话结论

> 这项最小实验的意义，不是证明大模型写得好，而是证明它是否值得被当作 persona teacher 使用。
