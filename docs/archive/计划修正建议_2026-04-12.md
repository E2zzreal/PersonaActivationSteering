## 1. 是否需要修正计划

**结论：需要。**

但不是推翻原计划，而是要做一次**目标重排和优先级重置**。

原因很明确：

1. 原计划的核心假设是“只要把配置贯通、训练链路打通，PersonaSteer 有望优于 baseline”；
2. 最新训练评估报告显示，当前 **baseline（无注入）反而表现最佳**，且注入模型普遍存在明显的 **思考过程泄露**；
3. 这说明现在的主矛盾已经不只是“训练链路是否可信”，还包括：
   - **注入本身是否在破坏输出质量**；
   - **当前成功标准是否定义错误**；
   - **是否应先解决副作用再继续扩大训练投入**。

因此，原 P0 方案仍有价值，但它已经不足以完整覆盖当前问题面。

## 2. 最新事实对原计划的影响

## 2.1 已被新证据强化的原判断

以下原判断被最新结果进一步证实：

### A. 注入效果未被验证成立
`docs/training_evaluation_report.md:1` 给出的结论是：
- `baseline` 得分最高：`3.803`
- `stage3_v2` 为 `3.761`
- 其他注入版本也未超过 baseline

这说明：
- “PersonaSteer 当前未证明能提升个性化质量”这一判断成立；
- 原先把“验证是否优于 baseline”作为关键目标是正确的。

### B. 思考泄露已上升为核心阻塞问题
报告显示：
- `baseline` 思考泄露率：`1.3%`
- `stage3_v2` 思考泄露率：`34.0%`

这意味着注入模型不仅没赢，还显著放大了 undesirable behavior。这个问题的优先级现在应高于很多原本的结构优化问题。

## 2.2 被新证据改变的部分

### A. 原计划过于强调“把训练做完”
从现在看，继续盲目推进更多 stage 或更多训练组合，并不会自动逼近正确答案。因为主问题可能不是“没训练够”，而是“当前注入方向本身在诱发错误行为”。

### B. 原计划对“成功标准”的定义偏乐观
之前更关注：
- gate 是否生效；
- contrastive loss 是否非零；
- 主线是否统一。

这些都重要，但还不够。现在必须加入新的硬性成功标准：
- **注入模型不得显著高于 baseline 的思考泄露率**；
- **注入模型若无法超过 baseline，则默认不进入下一轮大规模训练**。

## 3. 计划修正方向

## 3.1 原 P0 方案保留的部分

以下任务仍然保留，且仍然必要：

1. `gate` 配置贯通；
2. contrastive loss 真正生效；
3. 主线配置收敛；
4. 统一评估协议。

因为这些仍然决定实验结论是否可信。

## 3.2 需要新增的 P0 / P1 任务

### 新增 P0-A：思考泄露止损任务

#### 目标
先把注入模型的明显副作用压住，否则所有评分比较都会被输出质量劣化污染。

#### 原因
思考泄露已经不是小瑕疵，而是影响用户观感和 Judge 评分的主问题。

#### 建议动作
1. 为评估和推理加入泄露检测指标；
2. 增加后处理过滤或 stop pattern 作为临时止损；
3. 抽样定位哪些 persona / stage / prompt 最容易诱发泄露；
4. 区分“训练学坏了”和“生成配置触发了推理模式”。

### 新增 P0-B：重新定义实验门槛

#### 目标
防止继续为无效或负收益模型投入训练资源。

#### 新门槛建议
任一注入模型若满足以下任一条件，则不进入下一轮扩大实验：
- 评分未超过 baseline；
- 思考泄露率显著高于 baseline；
- 输出质量出现机械化或明显不自然退化。

### 新增 P1-C：注入副作用归因分析

#### 目标
区分问题到底来自哪里：
- 注入强度过大/过小；
- 注入层选错；
- v_t 学到错误模式；
- 训练数据带有隐式 CoT 倾向；
- 生成参数放大了推理痕迹。

#### 建议产物
- 不同 stage 的 gate 分布与泄露率对照表；
- 不同配置的样本级错误归因表；
- `baseline` vs `injected` 的典型失败案例集。

## 4. 更新后的优先级

### 新 P0（立即处理）
1. **P0-A：思考泄露止损与量化监控**
2. **P0-B：建立“必须优于 baseline 才继续”的实验门槛**
3. **原 T1：贯通 gate 配置参数**
4. **原 T3/T4：让 contrastive 训练真正生效**

### 新 P1（P0 完成后）
5. 主线配置与目录收敛
6. 注入副作用归因分析
7. 重新设计 Stage 3 的目标或损失权重

### 可降级项
- 单纯增加并行训练能力
- 扩大实验矩阵
- 新 persona 配置扩展

这些在当前阶段不应优先于“证明注入是否值得保留”。

## 5. 对当前实施文档的修订建议

## 5.1 `docs/P0修复方案设计文档.md`
建议补充：
- 新增“思考泄露止损”章节；
- 新增“baseline 不可被恶化”的成功标准；
- 将“更多训练”从默认动作改为条件动作。

## 5.2 `docs/P0实施任务清单.md`
建议调整：
- 在 T1 前增加一个新的 **T0：建立泄露监控与止损机制**；
- 在 T7 前增加一个新的 **T6.5：baseline 门槛判定**；
- 将“若未超 baseline 则暂停扩大训练”写入验收标准。

## 6. 一句话结论

> 现在不是单纯“把 PersonaSteer 训练做完”的阶段，而是“先证明注入不会比 baseline 更差”的阶段；因此计划必须从“打通训练链路”升级为“先止损、再归因、再决定是否继续扩大投入”。
