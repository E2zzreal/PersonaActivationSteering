## 3. 直接结论

### 3.1 M1 审计判断被强力验证

M1 的判断是：
- baseline 与 injected 推理链路未对齐；
- 这种差异可能显著放大 injected 泄露率。

本轮结果强力支持这一点：
- stage3 泄露率从 `30.00%` 下降到 `2.22%`；
- 平均分从 `2.833` 提升到 `3.567`；
- 变化幅度远大于此前 early / mid / late 的层位差异。

这意味着：

> 之前 observed 的 injected 高泄露，主要并不是“注入机制天然必然泄露”，而是被不对齐的推理链路显著放大了。

### 3.2 当前 stage3 已基本接近 baseline 泄露水平

对比本轮：
- baseline：`0.00%`
- stage3：`2.22%`

这说明：
- 在推理链路对齐后，stage3 仍有轻微残余泄露；
- 但已经从“系统性大面积 planning 泄露”降到了“少量边缘残余”。

### 3.3 注入模型的能力判断需要重新评估

本轮修复后：
- stage3 平均分 `3.567`
- baseline 平均分 `3.522`

虽然这只是 90 条样本的小规模结果，但它已经说明：
- 之前“baseline 一定优于 injected”的结论需要更谨慎；
- 至少在对齐推理链路后，stage3 已经具备重新进入有效竞争的资格。

## 4. 残余泄露特征

本轮 stage3 只剩 2 条命中，且特征明显变轻：
- 一条命中 `I need to`，出现在正常内容句中；
- 一条命中 `let me think`，属于短促口语化插句；
- 已不再是早期那种大段 `Okay, the user is... / I need to respond...` 的完整 planning 泄露。

说明：
- 当前主要问题已从“系统性 planning 外显”
- 降级为“少量残余口语化 thinking-like 表达”。

## 5. 对计划的修正

基于本轮结果，当前计划应做如下调整：

1. **M1 / M1.5 结论成立，且优先级目标已达成**；
2. 不再把“高泄露”作为 injected 的主判断标签；
3. 后续主线应转向：
   - 用统一链路重新评估 baseline / stage1 / stage2 / stage3；
   - 重写当前权威结论与报告；
   - 再决定是否还需要继续推进训练侧约束修正。

## 6. 一句话结论

> M1.5 结果表明，当前 injected 高泄露问题大头并不是模型本体必然外显 thinking，而是由不对齐的推理链路显著放大；在对齐到 chat template + `/no_think` + `enable_thinking=False` 后，stage3 泄露率已从 30.00% 降到 2.22%，并且平均分反超同轮 baseline。
