# M1.5 推理链路对齐修复（2026-04-13）

日期：2026-04-13
最后验证日期：2026-04-13
状态：已实现，待重跑验证

## 1. 修复目标

本次修复只做最小范围调整：
- 不改训练主逻辑；
- 不改模型结构；
- 不改 loss；
- 仅把 injected 评估/推理入口尽量对齐到 baseline 口径。

## 2. 本次改动

修改文件：
- `scripts/v4_eval_qwen3.py`

### 2.1 injected 改为使用 chat template

修复前：
- 使用手写 prompt：`用户: ...\n助手:`

修复后：
- 与 baseline 一样，使用：
  - system + user messages
  - `tokenizer.apply_chat_template(...)`

### 2.2 injected 显式加入 no_think 控制

修复后 injected 与 baseline 一样：
- system 中加入 `/no_think`
- `enable_thinking=False`（若 tokenizer 支持）

### 2.3 injected 改为只解码新增 token

修复前：
- 直接解码整段，再按 `"助手:"` 切分

修复后：
- 记录 `prompt_len`
- 只解码 `generated_ids[0][prompt_len:]`

### 2.4 decode 参数保持一致

当前 injected 对齐为：
- `max_new_tokens=150`
- `temperature=0.7`
- `top_p=0.9`

## 3. 预期影响

若 M1 审计判断成立，则本次修复后应出现以下趋势之一：

1. injected 泄露率明显下降；
2. injected 分数略有回升；
3. 至少能排除一大块由 prompt / template / no_think 不一致引入的偏差。

## 4. 不在本次修复范围内

- `PersonaSteerModel.generate(...)` 自定义逐 token 解码路径仍保留；
- baseline 与 injected 仍不是完全同一 runtime；
- 训练目标与输出形式错位问题仍未处理。

因此，本次只能视为：
- **M1 的最小闭环修复**，不是终局修复。

## 5. 建议重跑

推荐先只重跑两组：
- baseline
- stage3

对比修复前后：
- `avg_score`
- `thinking_leak_rate`
- `clean_thinking_leak_rate`

## 6. 一句话结论

> M1.5 的意义在于先消除最明显的推理链路偏差；只有在 injected 也走 chat template + `/no_think` + `enable_thinking=False` 后，后续对 loss、注入层或注入强度的判断才更可信。
