## 1. 文档目的

本文总结 PersonaSteer 项目截至当前阶段的实验现状、关键证据、已完成工作、阶段性结论，以及后续最值得推进的方向。

## 2. 当前结论总览

### 2.1 总结一句话

> 当前 PersonaSteer 注入模型尚未证明能稳定优于 baseline；其主要问题不是训练数据普遍含伪思考，而更像是在“训练目标提升空间有限 + gate 几乎饱和 + 注入层位置可能不理想”的共同作用下，诱发了明显的思考过程泄露。

### 2.2 当前最重要的判断

1. **baseline 仍是当前最强参考**；
2. **注入模型存在显著思考泄露**；
3. **训练数据显式伪思考污染极低**，不能作为主要根因；
4. **训练监督目标相对 baseline 有一定优势，但不够压倒性**；
5. **gate 对 leak / non-leak 的区分能力几乎没有，且整体接近饱和**；
6. 下一步最值得优先分析的是：**注入层位置是否是主要副作用来源之一**。

## 3.2 baseline 与注入模型对比结论

这表明：
- 注入模型当前没有带来明确收益；

## 3.4 训练数据伪思考审计

已完成：
- `scripts/audit_thinking_in_training_data.py`
- 对 `data/split/train.jsonl` 和 `data/split/val.jsonl` 进行了审计

结果：
- train：`27770` turn 中命中 `41` 条，比例 `0.0015`
- val：`6610` turn 中命中 `10` 条，比例 `0.0015`

解释：
- 当前训练/验证数据中的显式伪思考比例极低；
- 命中样本多数只是正常表达，如 `I should try...`、`I need to unwind...`；
- 不足以解释 injected 模型中的系统性 planning 泄露。

结论：
- **不支持“训练数据普遍污染导致泄露”的主假设。**

## 3.5 训练集监督质量审计

已完成：
- `scripts/audit/prepare_training_quality_audit.py`
- `scripts/audit/judge_training_quality_audit.py`
- 对 50 条训练样本进行了 `gold vs baseline` A/B 审计

结果：
- 总样本：`50`
- gold 胜：`27`
- baseline 胜：`15`
- error：`8`
- 原始 gold 胜率：`0.54`
- 排除 error 后有效样本 gold 胜率约：`64.29%`

解释：
- gold 相对 baseline **有优势**；
- 但优势 **不算压倒性**；
- 说明训练监督目标存在一定提升空间，但上限并不特别高。

结论：
- **训练目标有价值，但不够强。**
- 这意味着注入模型即使优化成功，也未必天然拥有很大的超过 baseline 的空间。

## 3.6 Gate 与泄露归因

已完成：
- 在 `scripts/v4_eval_qwen3.py` 中加入样本级 gate 统计输出
- 新增 `scripts/analyze_gate_leak_correlation.py`
- 对 `stage3_v2` 结果做 gate × leak 归因分析

结果：
- `gate_mean_leak = 0.96393`
- `gate_mean_non_leak = 0.964164`
- `gate_max_leak = 0.999995`
- `gate_max_non_leak = 0.999995`
- `layer_mean_leak` 与 `layer_mean_non_leak` 几乎重合

解释：
- leak / non-leak 样本的 gate 分布几乎没有区别；
- gate 当前对样本的动态区分能力极弱；
- 从数值看，gate 基本接近全层饱和。

结论：
- **当前不能支持“某些泄露样本被更强注入”这一假设。**
- 更合理的解释是：
  - gate 对所有样本都几乎全开；
  - 真正更值得怀疑的是注入层位置本身。

## 4. 当前综合判断

将以上证据串起来，当前最合理的解释是：

1. 训练数据本身并没有普遍教模型输出伪思考；
2. 训练监督目标比 baseline 略优，但上限有限；
3. gate 几乎饱和，难以发挥真正的动态门控作用；
4. 在这种情况下，如果注入层位置选得不理想，就容易把模型稳定推向“规划/推理子空间”；
5. 最终表现为：
   - 思考泄露明显增加；
   - 但人格化收益没有明显超过 baseline。

## 6. 当前不建议优先做的事

基于现有证据，当前不建议优先做：

1. 继续大规模扩大训练矩阵；
2. 直接把训练数据大规模清洗为主线任务；
3. 继续堆更多保守型输出清洗规则；
4. 在未完成层位置归因前就大改 loss。

## 7. 当前最值得推进的下一步

### 第一优先
- **B1：注入层位置归因分析**

目标：验证当前 `[8..15]` 是否天然更容易诱发 planning / reasoning 泄露。

### 第二优先
- 若层位置问题不显著，再回头讨论：
  - gate 为什么整体饱和；
  - loss 如何间接鼓励外显 planning；
  - 是否需要更强的训练目标约束。

## 8. 一句话结论

> 当前 PersonaSteer 的主要瓶颈已经从“数据里有没有明显思考污染”转移到“有限的训练目标提升空间 + 饱和的 gate + 可能不理想的注入层位置”这组三因素；下一步最应优先验证的，是注入层位置是否正把模型系统性推向 reasoning 子空间。
