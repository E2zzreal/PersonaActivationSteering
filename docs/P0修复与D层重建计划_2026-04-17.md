## 二、P0 修复（已完成）

### P0-1：gate 配置参数未贯通

**问题**：`gate_init_bias` / `gate_max` 在 `injection.py:129-135` 硬编码，YAML 配置值完全无效。所有 gate 实验实际使用同一组参数（-2.0 / 1.0），过去的 gate_init 系列实验结论需要重新审视。

**修复**：贯通 4 层传递链

```
YAML training.gate_init_bias / gate_max
  → scripts/train.py: training_config.get("gate_init_bias", -2.0)
  → PersonaSteerConfig: 新增 gate_init_bias / gate_max 字段
  → PersonaSteerModel.__init__ / set_backbone: 传入 SteeringInjection
  → SteeringInjection.__init__: 接受参数并传给 DynamicGate
  → DynamicGate: 已支持，无需修改
```

**涉及文件**：
- `src/models/persona_steer.py`：PersonaSteerConfig 新增两字段；两处 SteeringInjection 创建传参
- `src/models/injection.py`：SteeringInjection.__init__ 增加参数
- `scripts/train.py`：读取 training_config 传入 PersonaSteerConfig

### P0-2：SCL 因 batch_size=1 永远返回 0

**问题**：Stage2/3 配置 `batch_size=1`，`SupervisedContrastiveLoss.forward` 在 `losses.py:96` 显式判断 `batch_size <= 1` 返回零损失。`scl_weight=0.1` 配置完全无效。

**分析**：ALOE 数据集有 57 个唯一 personality，每个出现 48-61 次，适合按 personality 分组采样。

**修复**：实现 `PersonalityGroupedSampler`，并在 Stage2/3 中启用

- **新文件** `src/data/grouped_sampler.py`：
  - 按 personality 字符串分组，建立 personality → indices 映射
  - 每个 batch 全部来自同一 personality，保证正例对存在
  - 支持 epoch-level shuffle，`set_epoch()` 接口
  - 验证：batch_size=4 时，697 个 batch，每 batch 100% 同 personality

- **train.py 修改**：检测 `scl_weight > 0` 时自动切换为 grouped sampler

- **配置修改**：
  - `configs/train_stage2_qwen3.yaml`：batch_size 1 → 4
  - `configs/train_stage3_qwen3.yaml`：batch_size 1 → 4

## 三、D 层数据重建计划

### 目标

将训练数据的人格对齐分上限从 ALOE 的 2.6 提升至接近 Claude 水平（3.5+），使超网络能学到更强的 personality → 行为映射。

### Phase D1：扩大并行对话生成（进行中）

**脚本**：`scripts/audit/generate_parallel_dialogues.py`

**参数**：
```bash
python scripts/audit/generate_parallel_dialogues.py \
  --aloe_input data/split/val.jsonl \
  --output results/parallel_dialogues/dialogues_full.json \
  --n 661 --max_turns 5 --device cuda:2 --timeout 120
```

**状态**：后台运行中，已完成 42/661（预计约 20 小时）。支持断点续传。

**输出格式**：每条对话包含三源并行回复（claude_response / qwen3_response / aloe_response），基于相同的 ALOE personality 和用户输入。

### Phase D2：构建 SFT 训练集

**脚本**：`scripts/build_claude_sft_data.py`（已完成）

**用法**：

```bash
# 基础用法：Claude 回复 → SFT 格式
python scripts/build_claude_sft_data.py \
  --input results/parallel_dialogues/dialogues_full.json \
  --output data/claude_sft/train.jsonl \
  --source claude

# 同时构建 DPO 偏好对（preferred=claude, rejected=qwen3）
python scripts/build_claude_sft_data.py \
  --input results/parallel_dialogues/dialogues_full.json \
  --output data/claude_sft/train.jsonl \
  --source claude \
  --build_dpo --dpo_rejected qwen3 \
  --dpo_output data/claude_dpo/train.jsonl
```

**输出格式**：与 `data/split/train.jsonl` 完全兼容（`user_id / profile / personality / conversations`），可直接被 `ALOEDataset` 加载。

### Phase D3：Stage1 SFT 重训

依赖 D2 完成后执行：

```bash
python scripts/train.py \
  --config configs/train_stage1_qwen3.yaml \
  --data data/claude_sft/train.jsonl \
  --output checkpoints/stage1_claude_sft
```

**关键验证指标**：训练损失 < ALOE 基线的同期损失；生成回复不出现 thinking leak。

### Phase D4：评估对比

用三源评估管道（`scripts/audit/score_three_sources.py`）对新 Stage1 模型评分：

**通过标准**：新模型人格对齐分 > 2.833（Qwen3 baseline），证明 Claude 数据有效提升训练上限。
