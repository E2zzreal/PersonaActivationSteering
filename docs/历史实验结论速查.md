## B. 注入层位置归因（B1，2026-04-13）

**结论**：`[8..15]` 不是好默认值，`[4..11]` 更优。

| 窗口 | 评分趋势 |
|------|------|
| `early_4_11`（前移） | 最优候选 |
| `mid_8_15`（当前默认） | 不推荐继续使用 |
| `late_16_23`（后层） | 最差 |

- 问题未被根治：层位置是影响因素之一，但不是唯一根因

## C. 训练数据质量（2026-04-13）

- 主要根因是：训练监督目标（ALOE gold）的人格对齐信号太弱
- Gate 接近饱和，区分能力有限

## F. D 层重建实验（2026-04-19，D4 评估）

**实验**：661 条 Claude 多轮对话替换 ALOE 作为 Stage1 SFT 训练数据（inject_layers=[4..11]）

| 来源 | GPT-5.4 均分 | 对比 baseline |
|------|---:|---:|
| Claude-Opus-4.6 | 3.833 | +1.000 |
| **新模型（Claude SFT Stage1）** | **3.000** | **+0.167 ✅** |
| Qwen3-4B baseline | 2.833 | 基准 |
| ALOE gold | 2.600 | -0.233 |

**分数分布**：0×1分, 1×2分, 28×3分, 1×4分（几乎全部 3 分）

**结论**：
- 假设成立：Claude 数据确实有效，突破 baseline +0.167（+6%）
- "3 分墙"：模型学到了更好的对话，但缺乏人格不可替换的行为签名
- Judge 评语：「温暖友好自然，但换成任何外向型人格也成立」
- 2 分案例：出现身份混淆（模型编造了不属于人格的个人经历）

**下一步**：DPO 阶段（preferred=Claude, rejected=Qwen3，已有 3305 条偏好对）有望推动 3→4 分突破

## G. 三阶段完整管道实验（2026-04-20，661 条 Claude SFT 数据）

**管道**：Stage1(SFT) → Stage2(SFT+gate) → Stage3(SFT+gate+SCL)，inject_layers=[4..11]

| 模型 | 均分 | 分布 | 说明 |
|------|---:|------|------|
| Claude | 3.833 | — | 教师上限 |
| **Stage2** | **3.033** | 1×2, 27×3, 2×4 | **最佳配置** |
| Stage1 | 3.000 | 1×2, 28×3, 1×4 | SFT only |
| Stage3 (scl=0.1) | 3.000 | 30×3 | SCL 贡献太小 |
| Stage3 (scl=1.0) | 2.967 | 1×2, 29×3 | SCL 反而有害 |
| baseline | 2.833 | — | Qwen3-4B 无注入 |

### SCL 三层 bug 修复历程

| Bug | 原因 | 修复 |
|-----|------|------|
| batch_size=1 → SCL=0 | 单样本无对比 | batch_size=4 + grouped sampler |
| user_ids 唯一 → pos_mask=0 | trainer 传 user_ids，SCL 优先用 user_ids 匹配 | 改为用 personality 字符串匹配 |
| 全部同 personality → 无负例 | grouped sampler 整 batch 同 personality | 混合 batch：2 personality × 2 samples |

三层 bug 全部修复后，SCL 仍然无效（Δ=0.003/6 epochs），根因：

### v_t 不分化的根本原因

- 冻结 encoder（Qwen3-4B backbone）对不同 personality 文本的 embedding 差异极小
- projection MLP 无法放大微小的 embedding 差异
- gate_init_bias=-2.0（注入强度 12%）进一步压缩 v_t 对输出的影响
- 结果：所有 personality 的 v_t 几乎相同，SCL 无梯度可优化

### DPO 同样失效

DPO 依赖 token 级 logprob 差异，但 12% 注入强度导致 80% 步骤 margin=0（模型和参考模型 logprobs 相同）。

### 突破方向（等全量 2777 条数据）

1. 增大 gate_init_bias（-2.0 → -0.5，注入强度 12% → 38%）
2. 数据量 ×4（661 → 2777），减少过拟合
3. 或重新设计 HyperNetwork 使其更好地区分 personality

## H. 瓶颈诊断实验（2026-04-20）

**目的**：确定 3 分墙的根因是 encoder 相似度还是 gate 注入强度。

**预诊断**：14 个 personality 的 encoder embedding 余弦相似度 **均值 0.959**（几乎无区分度）。

| 实验 | 改动 | 均分 | gate_entropy |
|------|------|---:|---:|
| A1 (Embedding Table) | 14 个可学习向量替换 encoder | 3.000 | **-1.81** |
| B (gate=0.0) | 注入强度 12% → 50% | 3.000 | -2.08 |
| Stage2（对照） | 原始配置 | 3.033 | -2.07 |

**关键发现**：

- ❌ **encoder 不是瓶颈**：A1 用完美分化的 embedding + 更活跃的 gate，仍然 3.0
- ❌ **gate 强度不是瓶颈**：B 用 50% 注入，仍然 3.0
- ✅ **注入机制本身工作正常**（gate_entropy 在 A1 中显著不同，证明 gate 能响应不同输入）

**结论**：3 分墙的根因不在注入机制（I 层），而在数据质量（D 层）：
- Claude 训练数据中的人格签名密度不足（Claude 自身 30 条也大量 3 分）
- SFT 只教「怎么说」不教「为什么这个 persona 必须这么说」
- 661 条 × 14 personality ≈ 47 条/personality，样本量不足以学到精细人格差异

**后续方向**：
1. 升级 Claude 生成 prompt（加入 REQUIRED/FORBIDDEN 行为约束，提高训练数据的人格签名密度）
2. 等全量 2777 条数据 + 升级后的 prompt 重新生成
3. 探索对比训练目标（同一用户输入，不同 persona 回复的 pair-wise 对比）
