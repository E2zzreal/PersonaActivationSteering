## 1. 目的

统一训练结果、评估结果、分析报告三者之间的引用口径，避免出现：
- 报告引用 A 目录；
- 判定脚本读取 B 目录；
- 人工讨论基于 C 目录。

## 2. 口径判定原则

### 2.1 权威性排序
当存在多套结果时，按以下顺序确定“当前权威口径”：

1. **最新 Git 提交对应的分析报告引用结果**
2. **时间更晚、覆盖更多配置、与报告一致的结果目录**
3. **若报告未显式写明目录，则以最近生成且结构完整的结果目录为准**
4. **旧目录一律视为历史参考，不再作为当前主结论依据**

### 2.2 当前生效口径
根据 Git 时间线：
- `3ee2d04`：新增训练与评估综合分析报告
- `755944d`：更新评估报告并加入 baseline 对比分析

因此，`docs/training_evaluation_report.md` 视为**当前最新分析报告**。

当前报告中的关键口径应由以下目录共同支撑：
- baseline 结果目录：`results/eval_baseline_20260412_180724/`
- stage3_v2 结果目录：`results/eval_stage3_20260412_151649/`
- 其他 stage1/stage2 对比若未重新生成，则明确标注来源目录和时间

## 3. 强制规则

1. 任一分析报告必须在文首或附录明确写出所引用的结果目录；
2. baseline 门槛判定脚本必须使用与分析报告同一口径的数据；
3. 若结果来自多目录拼接，必须在文档中逐项列明；
4. 未被当前报告引用的旧结果目录一律归档到 `results/archive_20260412/legacy_eval/`；
5. 不允许再默认把 `results/` 下最先看到的 `summary.json` 当作当前权威结果。
